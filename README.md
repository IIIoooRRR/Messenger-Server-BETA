# Messenger-Server-BETA-v1.0-
Легковесный мессенджер с асинхронной архитектурой на WebSocket и Kafka.  
Сервер не хранит историю сообщений — всё на клиенте.

## 🚀 Технологии
- Джава 17
- Spring Boot 3.4.2 (Web, JPA, WebSocket, Validation)
- Kafka — буфферизация(пока в процессе разработки), ассинхронность сообщений
- PostgreSQL / H2 — хранение пользователей
- Lombok — уменьшение количества дефолтного кода
- Сборка на Мавен
- Jakarta validation - для уменьшения количества обработки ошибок. Все поставлено при приеме данных на сервер

  ## 📌 Особенности
- Сообщения не хранятся на сервере* - история только на клиенте
- ID генерируется из номера (MD5 + соль при коллизиях)
- Асинхронность - WebSocket не блокируется

  # Принцип работы:
### 1. Регистрация пользователя
`POST /add/user` — сервер генерирует ID на основе номера (MD5), сохраняет в БД.

### 2. WebSocket подключение
Клиент шлет заголовок `id` при коннекте.  
Сервер:
- Моментально освобождает сетевой поток
- Кидает задачу в **пул №1** (разбор события)
- Ставит статус `ONLINE` через **пул №2** (работа с БД)

### 3. Отправка сообщения
`@MessageMapping("/chat/send")` → Kafka (`user-mess` topic)  
Консьюмер забирает → `SimpMessagingTemplate` отправляет получателю

## 🐞 Обработка ошибок

- `UserNotFound` (404) — если пользователя нет
- `MessageNotFound` (404) — заглушка на будущее
- Валидация входных данных через `@Valid`

## 📦 Запуск

```bash
# Запустить Kafka (если локально)
# или настроить в application.properties
spring.kafka.bootstrap-servers=localhost:9092

# Запустить приложение
./mvnw spring-boot:run
```
